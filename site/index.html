Code Agent Runtime

An independent systems/research platform for the runtime infrastructure around coding agents: task setup, tools, permissions, execution, tracing, replay, scoring, regression comparison, and reports.

status: Milestone 3 — tool registry and core tools

Brief explanation

Modern coding agents need a runtime, not just a prompt. This project builds the controlled environment that makes coding-agent behavior reproducible, inspectable, and evaluable: versioned tasks, structured tools, permission policies, command execution, trace recording, replay, scoring, and reports.

Deeper technical explanation

A run takes a versioned task and an agent and produces a scored, replayable trace:

task -> workspace -> agent loop -> tools -> tests -> scoring -> report
                         \__ trace recorder captures every step __/

Deterministic mock and scripted agents are first-class so the whole pipeline can be exercised without any model call. Optional, cost-guarded Claude API / Claude Code adapters are added only after the runtime foundations exist.

Current test/result status

CPU-only tests pass (python3 -m pytest -q), including environment, hygiene-scanner, task-format, and tool tests (per-tool unit tests plus an integration test that fixes a fixture bug end to end) and a guard that asserts the committed tree is clean. No evaluations have been run; see Results.

Open questions / limitations

No sandbox or evaluations are implemented yet. This is not a production-grade sandbox and not a benchmark leaderboard. See Limitations.
