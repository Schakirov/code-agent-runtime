Limitations & non-goals

What is true today, and what this project deliberately is not.

status: current as of Milestone 1

Brief explanation

At Milestone 1 the repository has a package, CLI, docs, website, packaging, CPU-only tests, an environment check, and a repository hygiene scanner. The agent runtime, tools, sandbox enforcement, tracing, and scoring are not built yet.

Deeper technical explanation

Known limitations

  • No sandbox. No enforced runtime isolation — see Security. The hygiene scanner is a commit-time guard, not a runtime control.
  • Hygiene secret detection is heuristic. High-precision patterns favor avoiding false positives over catching every secret; it can miss novel or obfuscated credentials and is not a guarantee.
  • Hygiene scope is tracked files. Only git ls-files content is scanned by default; bare env/ENV dirs are not treated as virtualenvs. Use --include-untracked to widen scope.
  • No evaluations. Results is intentionally empty.
  • No model integration. No LLM calls or Claude adapters yet.
  • Environment assumption. Tests assume python3 3.10+ with pytest; the src/ layout plus conftest.py shim runs the suite without an install.

Non-goals

Not a production-secure sandbox; not a replacement for any commercial coding agent; not a claim of beating commercial agents; not a benchmark leaderboard; not a large-scale paid-API evaluation; not a web product; not a tutorial.
