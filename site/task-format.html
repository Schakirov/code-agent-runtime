Task format

Versioned, declarative task definitions that pin everything a run needs.

status: planned (Milestone 2) — not implemented yet

Brief explanation

A task is a small, version-controlled file describing one unit of work: what to do, which fixture repo to do it in, what tools are allowed, and how success is measured.

Deeper technical explanation

Planned task fields:

FieldMeaning
id, title, versionStable identity and schema version.
promptInstruction given to the agent.
fixture pathRepo snapshot the workspace is built from.
allowed toolsSubset of the tool registry the agent may use.
timeout, resource limitsBounds on runtime and output.
scoring method, test commandHow pass/fail is determined.
metadataTags such as bugfix, refactor, CLI change, security.

Valid tasks load cleanly; invalid tasks must produce readable errors. The CLI will be able to list and inspect tasks.

Current implementation status

Not implemented. The schema and example tasks (bugfix, CLI change, unsafe-command demo) arrive in Milestone 2.
